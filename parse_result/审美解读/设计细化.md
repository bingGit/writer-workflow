# 审美解读项目 - 设计细化

## 一、解读的话术风格设计

### 1.1 核心原则：朋友语气为主，术语为辅

**策略**：先用朋友的语气让用户"感受到"，再用术语帮用户"说得出"

| 维度 | 推荐结构 |
|------|---------|
| 主体 | 用口语化的方式解读（"你注意到了吗？光从左边洒进来..."） |
| 附带 | 括号或小字注明术语（这叫"侧光/伦勃朗光"） |

### 1.2 话术结构模板

**四段式解读**：

```
1. 观察（我看到了什么）
   "这张照片里，光从左边斜着打过来，在人物脸上形成了明暗分界线..."

2. 感受（这让我觉得）
   "这种光让整个画面有一种戏剧感，人物的轮廓被勾勒得特别清晰..."

3. 术语标注（这个手法叫）
   "摄影里把这种光叫做**伦勃朗光**，因为17世纪的画家伦勃朗特别喜欢用这种打光方式。"

4. 启发（你可以试试）
   "下次在室内拍人像时，试试让光从侧面45度打过来，你的照片也能有这种感觉。"
```

### 1.3 术语植入方式

| 方式 | 示例 |
|------|------|
| **括号附注** | "这种'留白'（摄影里叫**负空间**）让画面会呼吸" |
| **比喻+术语** | "你的眼睛会被这条线一路带过去——这就是所谓的**引导线**" |
| **故事化** | "摄影师管这个叫**三分法**，据说从古希腊就开始用了" |
| **反问式** | "你有没有发现，主体不在正中间反而更舒服？这叫**黄金分割**" |

### 1.4 情感词汇 + 术语对照表

| 朋友语气表达 | 对应术语 |
|-------------|---------|
| "画面很有张力、很抓眼球" | 高对比度 |
| "颜色很克制、有一种静谧感" | 低饱和度 |
| "留出的空白让画面会呼吸" | 负空间 |
| "你的眼睛会自然被吸引到这里" | 黄金分割 / 趣味中心 |
| "这条线把你的视线一路带到...那里" | 引导线 |
| "背景被虚化，人物更突出" | 浅景深 / 大光圈 |
| "光从侧面打过来，脸上有明暗" | 侧光 / 伦勃朗光 |

### 1.5 避免的雷区

- **❌ 只有术语**："构图使用了三分法，色温约4500K"
- **❌ 完全没术语**：用户听完还是不知道该怎么描述
- **❌ 术语先行**："这是伦勃朗光，就是那种..."
- **✅ 正确顺序**：先感受 → 再命名 → 最后应用

## 二、用户成长体系设计

### 2.1 成长阶梯模型

```
Level 1: 看见（Awareness）
  └─ 能力：知道照片有好坏之分
  └─ 产品动作：展示对比（"你觉得哪张更舒服？"）

Level 2: 理解（Understanding）
  └─ 能力：能说出"为什么好看"
  └─ 产品动作：解读后让用户复述（"你觉得这张好在哪？"）

Level 3: 识别（Recognition）
  └─ 能力：能在新照片中发现同样的元素
  └─ 产品动作：挑战题（"这张照片用了类似的手法吗？"）

Level 4: 应用（Application）
  └─ 能力：能在自己的拍摄中运用
  └─ 产品动作：拍摄任务（"试着拍一张侧光的照片"）
```

### 2.2 成长可视化

- **审美积累值**：每次解读后 +10，用户可以看到自己的"审美经验值"
- **能力雷达图**：构图感 / 光影感 / 色彩感 / 情绪感
- **里程碑徽章**：
  - "初见光影"（完成第一次光线解读）
  - "构图眼"（识别10张照片的构图）
  - "情绪捕手"（理解5种情绪表达方式）

### 2.3 学习闭环

```
上传照片 → AI解读 → 用户思考 → 用户回答 → 反馈修正 → 能力增长
    ↑                                              ↓
    └──────────── 下一张照片 ←────────────────────┘
```

**关键点**：不是"我告诉你"，而是"你发现了什么？"

## 三、如何在产品中体现"人的价值"

### 3.1 三个不可委托的层面

| 层面 | 具体体现 | 产品设计 |
|------|---------|---------|
| **目标定义** | 什么是"好的解读" | 解读风格、深度、温度由人设定 |
| **质量把关** | 哪些解读可以展示 | 人工筛选典型案例、调整prompt |
| **认知引导** | 用户应该学到什么 | 设计成长路径、选择教学重点 |

### 3.2 "人味"植入点

1. **开场白变化**
   - 不是每次都"我来分析一下"
   - 而是："今天这张有点意思"、"我注意到一个细节"、"这张让我想起了..."

2. **个人观点标注**
   - 在AI分析基础上，加入"我的想法"栏目
   - 例："AI说这是三分法，但我觉得更有趣的是..."

3. **故事化案例**
   - 不是冷冰冰的"示例照片"
   - 而是"我上周在杭州拍的这张，当时光刚好..."

4. **失败分享**
   - 定期分享"我拍砸的照片"
   - 展示真实的学习过程，建立信任

### 3.3 人机分工矩阵

| 任务 | AI负责 | 人负责 |
|------|--------|--------|
| 技术分析 | 识别构图类型、光线方向 | 决定如何表达这些分析 |
| 情感解读 | 生成初稿 | 校准情感表达是否准确 |
| 学习建议 | 根据规则生成 | 设计规则、筛选建议 |
| 用户互动 | 自动回复常见问题 | 处理复杂情感诉求 |

### 3.4 差异化护城河

> 你的竞争优势不是"AI分析得准"，而是"你帮用户理解得透"。

- **AI是工具**：它帮你放大能力
- **你是灵魂**：你决定用户学到什么
- **产品是桥梁**：连接你的审美理解和用户的成长需求
