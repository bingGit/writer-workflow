# 人AI协作准则 - 写作思路

## 1. 核心因果链提取

从模型树中提取的核心逻辑：
> 工具能力↑ → 人的"输入质量"要求↑↑ → 否则陷入"高效的平庸"

核心公式：
> 创作力 = 独特输入 × 工具放大倍数
> 如果独特输入=0，工具再强也是 0×∞=无意义

## 2. 读者心理路径设计

**目标读者状态**：正在大量使用AI，但隐隐感到"哪里不对"
**目标情绪轨迹**：好奇 → 共鸣（被戳中） → 反思 → 找到方向

| 阶段 | 读者心理 | 文章对应 |
|------|---------|---------|
| 开头 | "AI确实很强" | 用个人经历切入——发现AI能写出"人味"后的震惊 |
| 共鸣 | "我也有这种感觉" | 点出"存在性恐慌"——我的价值在哪？ |
| 转折 | "原来问题在这" | 引出核心洞察：问题不是AI太强，而是你没定义好边界 |
| 方法 | "那我该怎么做" | 给出三问法工作流 |
| 升华 | "我想试试" | 留一个开放问题，邀请读者自己思考 |

## 3. 个人经历融入点

根据用户的独特性报告，需要融入的三个"独特标记"：

1. **痛苦现场**：发现AI能写出人味后的震惊
   - 融入位置：开头（建立共鸣）
   
2. **反共识判断**：不追热点，追本质
   - 融入位置：转折处（解释为什么这样思考）
   
3. **留白式结尾**：不给终极答案
   - 融入位置：结尾（抛出开放问题）

## 4. 正反对比设计

| 误区（❌） | 破局（✅） |
|-----------|-----------|
| 让AI定义目标（委托灵魂层） | 人定义目标，AI执行 |
| 追求"高效产出" | 追求"差异化输入" |
| 只要结果，不care过程 | 过程中建立掌控感 |

## 5. 文章结构骨架

```
1. 开头（150字）：一个让我困惑的瞬间
   - 个人经历：发现AI写的文章比我"更有人味"
   
2. 问题（150字）：这种困惑背后是什么
   - 存在性恐慌：如果AI都能模拟真实，我还剩什么？
   
3. 转折（200字）：重新定义问题
   - 问题不是"AI太强"，而是"你没定义好自己的边界"
   - 核心公式：创作力 = 独特输入 × 工具放大
   
4. 方法（300字）：如何找到自己的边界
   - 三个不可委托：目标、判断、责任
   - 三问工作流
   
5. 案例（100字）：减肥计划的类比
   - 直接用AI计划 vs 先想清楚"为什么要瘦"
   
6. 结尾（100字）：一个开放的问题
   - 如果未来AI也能提供"独特输入"，人的价值还剩什么？
```

## 6. 标题备选

- 《当AI比我写得更"真实"，我该怎么办？》
- 《AI时代，人的价值在哪里？》
- 《工具越强，你的边界越要清晰》
